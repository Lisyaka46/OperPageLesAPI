<br>Библиотека предоставляющая связь между программой/системой OperPageLes и вашим проектом.<br>
В библиотеке реализованы объекты интерфейса, которые используются в OPL.<br>
___Их также можно добавить в ваш проект для единого интерфейса.___<br><br>
API предоставляет доступ к таким функциям как:<br>
+ **Обработка форматированного текста BIU.**
> _Который разработан и используется внутри OPL_
+ **Управление JSON структурами**
> _Обработка данных файлов настроек или ярлыков в OPL_
+ **Реализация и компиляция собственных страничных приложений**
> _Функция которая позволяет добавлять собственные страничные приложения для OPL_<br>

## Как используется анимирование в OPL
Анимирование любого объекта интерфейса исполняется через его внутреннее свойство менеджера анимаций `MenegerAnimation`.<br>
Обработка анимирования происходит через статичную функцию класса `AnimateTakingZeroTo`/`AnimateTakingZeroFromTo` где передаётся сам менеджер, анимируемый элемент, его свойство, начальное и конечное/только конечное значения, время в структуре `TimeSpan`<br>
Есть возможность получить клон элемента анимации **WPF** через сам объект менеджера анимаций. Функция `GetCloneAnimationElementFromType` принимает входной тип который хранится и обрабатывается в менеджере: `DoubleAnimation`, `ThicknessAnimation` и т.п.
Если этот тип может обрабатываться менеджером то он вернёт ожидаемый тип объекта, инчае нет.

## Как работает обработка форматированного текста BIU
Обработка форматированной строки происходит посредством ключевых символв в тексте.<br>
BIU (Bold_Italic_Underline) работает посредством специального символа `%` который открывает форматирование строки, при написании `%%` символ % просто запишется в строку.<br>
После специального символа `%` может идти настройка фона для форматированного текста, она представляет 16-ричный вид `#FFFFFF`. Если цвет не нужен то эту настройку можно игнорировать<br>
Далее идут символы поределяющие каким должен быть текст:
+ `__` Жирный
+ `//` Курсив
+ `--` Зачёркнутый
> Данную настройку форматирования можно реализовывать вложенным способом.<br>
**Пример**:<br>
```%#FFFFFF__Этот текст белый и жирный %//Этот белый, жирный, курсивный//__%// А этот просто курсивный без цвета//```<br>

## Как работает обработка настроек
Настройки реализованы по принципу разделения на категории и на параметры.<br>
Категории представлены базовым абстрактным объектом `CategorySettingBase` в котором хранятся параметры.<br>
Базовый класс наследует объект `CategorySetting<T>`, где входной тип `T` ожидается `Enum` где ключи будут перечисленны числовым типом `uint`<br>
**Если `Enum` будет представлен ключами отличными от `uint`, то создание категории будет невозможным.**<br><br>

В категориях хранятся параметры, которые наследуют базовый класс `ParameterSettingBase`, который в свою очередь реализует интерфейс `IParameterSettingBase`.<br>
Этот интерфейс говорит о наличии значения по умолчанию в параметре и самого объекта значения.<br>
Параметры в настройках могут быть ограниченными и обычными.<br>
Обычный параметр категории настроек представлен типом `ParameterSetting<T>` с входным типом `T`, типом которого может быть любой объект.<br>
В отличии от обычных параметров, ограниченные параметры могут быть только числового типа: `int` и `double`.<br>
Они представлены соответствующими типами `LimitedParameterIntSetting` `LimitedParameterDoubleSetting`<br>
**При установке значения ограниченному параметру, значение будет подстравиваться под ограничения, не выдавая ошибку** `Math.Clamp`

В итоге в главном статичном классе `Setting` можно управлять, добавлять категории. В категориях можно добавлять параметры и управлять их поведением.<br>
Также в статичном классе `Setting` "и в категориях с параметрами" присутствуют события, через которые можно отслеживать добавление новой категории или параметра.<br>
Отслеживание изменения значения и т.д.

## Как работают языковые переводы
Языковые переводы реализуются через файл JSON в виде информации о переводе и сам перевод.<br>
Они могут быть в любом перечислении типа `Enum`. Ограничение лишь одно — перечисление должно быть в одной сборке с проектом.<br>
**В информацию о переводе входят такие свойства как:**<br>
+ Локальное имя `Locate`, на основе которого производится поиск отображения флага перевода. Подразумевается что этот ключ не будет повторяться в файле.
+ Версия перевода `Version`, строка указывающая на версию перевода. В системных языках используется версия _"Infinity"_, как характеристика о бесконечной поддержке перевода.
+ Название перевода `Name`, отображаемая строка имени языкового перевода.
+ Автор `Autor`, _название/ник/псевдоним автора_, который отображается в языковом переводе.
+ Контакты `Contacts`, это массив объектов со свойствами о контактах автора. Свойства для контакта автора лишь:<br>
Локальное отображение имени контакта `Locate`, ссылка на контакт автора `URL`, Маска отображаемой ссылки `Mask`.
+ Сам языковой перевод `Translation`. Он представляет собой объект который хранит в себе типы - категории, в которых уже хранятся параметры языкового перевода, которые могут быть использованы.<br>
Все подсловари данного параметра должны реализовывать `Enum` сохраняя ключи в `ULONG`, иначе данное перечисление не будет поддерживаемым и все его ключи будут записаны как лишние.

_Подсловари могут быть совершенно любого наименования — они отражают тип Enum : ulong. Ключи как раз являются названиями перечислений этого типа Enum._<br>
Если устанавливаемое страничное приложение будет использовать данную гибридную архитектуру языковых переводов.<br>
То есть возможность создавать свои типы перечислений, для обработки своих ключей языкового перевода в страничном приложении.

Здесь приведён базовый пример языкового перевода для двух подсловарей _"LangUITranslate", "LangSettingGeneralUITranslate"_<br>
Подсловарь `LangUITranslate` подразумевает использование для основного интерфейса OperPageLes.<br>
Подсловарь `LangSettingGeneralUITranslate` подразумевает использование для интерфейса общей категории настроек для OperPageLes.<br>
```
{
    "Locate": "Russian",
    "Version": "Infinity",
    "Name": "(Russian) Русский",
    "Autor": "Autor",
    "Contacts": [
        {
            "Locate": "Telegram",
            "URL": "https://t.me/OperPageLes",
            "Mask": "OPL Channel"
        }
    ],
    "Translation": {
        "LangUITranslate": {
           "HeadNameProgram": "NameProgram",
           "MainWindowTitle": "Главное меню"
        },
        "LangSettingGeneralUITranslate": {
           "GeneralTitle": "Общие"
        }
   }
}
```
