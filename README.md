# OperPageLes API [![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg)](https://opensource.org/licenses/Apache-2.0)
Библиотека предоставляющая связь между программой/системой OperPageLes и вашим проектом.<br>
В библиотеке реализованы объекты интерфейса, которые используются в OPL.<br>
___Их также можно добавить в ваш проект для единого интерфейса.___<br><br>
API предоставляет доступ к таким функциям как:<br>
+ **Обработка форматированного текста BIU.**
> _Который разработан и используется внутри OPL_
+ **Управление JSON структурами**
> _Обработка данных файлов настроек или ярлыков в OPL_
+ **Реализация и компиляция собственных страничных приложений**
> _Функция которая позволяет добавлять собственные страничные приложения для OPL_<br>

## Как используется анимирование в OPL
Анимирование любого объекта интерфейса исполняется через его внутреннее свойство менеджера анимаций `MenegerAnimation`.<br>
Обработка анимирования происходит через статичную функцию класса `AnimateTakingZeroTo`/`AnimateTakingZeroFromTo` где передаётся сам менеджер, анимируемый элемент, его свойство, начальное и конечное/только конечное значения, время в структуре `TimeSpan`<br>
Есть возможность получить клон элемента анимации **WPF** через сам объект менеджера анимаций. Функция `GetCloneAnimationElementFromType` принимает входной тип который хранится и обрабатывается в менеджере: `DoubleAnimation`, `ThicknessAnimation` и т.п.
Если этот тип может обрабатываться менеджером то он вернёт ожидаемый тип объекта, инчае нет.

## Как работает обработка форматированного текста BIU
Обработка форматированной строки происходит посредством ключевых символв в тексте.<br>
BIU (Bold_Italic_Underline) работает посредством специального символа `%` который открывает форматирование строки, при написании `%%` символ % просто запишется в строку.<br>
После специального символа `%` может идти настройка фона для форматированного текста, она представляет 16-ричный вид `#FFFFFF`. Если цвет не нужен то эту настройку можно игнорировать<br>
Далее идут символы поределяющие каким должен быть текст:
+ `__` Жирный
+ `//` Курсив
+ `--` Зачёркнутый
> Данную настройку форматирования можно реализовывать вложенным способом.<br>
**Пример**:<br>
`%#FFFFFF__Этот текст белый и жирный %//Этот белый, жирный, курсивный//__%// А этот просто курсивный без цвета//`<br>
